Event Time Placement

A separate "Scheduled" / "Event Time" field that shows when something will happen — distinct from the footer timestamp which shows when the alert was sent. Four placement options using a maintenance window scenario.

Option A — Amber Banner Below Subject RECOMMENDED
A warm amber strip sits between the subject and the FROM/TO meta row. Highly visible, clearly separate from the alert's own timestamp. Feels like a "sticky note" pinned to the alert.

Composer Field Design

  • Replaces the existing "Timestamp Override" field in the composer form
  • Label: "Event Time" with an OPTIONAL tag (same as FROM/TO)
  • Input: Two datetime-local inputs — "Start" and "End" (end is optional for single-point events)
  • When empty: No event time section appears on the card at all
  • Context labels: Auto-selects based on severity — MAINTENANCE → "Scheduled", ISSUE → "Started", INFO → "When", RESOLVED → "Duration"
  • The footer timestamp always shows the current time (when the alert was created) — no longer overridable
